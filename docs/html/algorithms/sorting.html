Sorting¶

Sorting algorithms are used to rearrange a given array according to a comparison operator on the elements.

pydsa.algorithms.sorting.bead_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Also known as Gravity sort, this algorithm was inspired from natural phenomena and was designed keeping in mind objects (or beads) falling under the influence of gravity.

pydsa.algorithms.sorting.bogobogosort(arr, /, *, key=<function <lambda>>, reverse=False, randomized=False)¶

Sort by recursively calling itself with smaller and smaller copies of the beginning of the list to see if they are sorted. (An algorithm that was designed not to succeed before the heat death of the universe on any sizable list.)

pydsa.algorithms.sorting.bogosort(arr, /, *, key=<function <lambda>>, reverse=False, randomized=False)¶

Sort by randomly swapping elements in the array, and hoping that it will eventually sort itself. Another way of implementation is to check all the permutations of the array until we find a sorted one.

pydsa.algorithms.sorting.bozosort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Sort by randomly swapping any two elements of the list and checking if the list is sorted.

pydsa.algorithms.sorting.bubble_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Sort by repeatedly swapping the adjacent elements if they are in wrong order.

pydsa.algorithms.sorting.bucket_sort(arr, /, *, key=<function <lambda>>, reverse=False, sorting_algorithm=<function insertion_sort>)¶

Sort by distributing the elements of an array into several buckets. Each bucket is then sorted individually, either using a different sorting algorithm, or by recursively applying Bucket sort.

pydsa.algorithms.sorting.cocktail_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

A variation of Bubble Sort, sort by traversing through a given array in both directions alternatively.

pydsa.algorithms.sorting.comb_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Improve on Bubble sort by using gap of size more than 1.

pydsa.algorithms.sorting.counting_sort(arr, /, *, reverse=False)¶

Sort by counting the number of objects having distinct key values.

pydsa.algorithms.sorting.gnome_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Similar to Insertion sort in that it works with one item at a time but gets the item to the proper place by a series of swaps.

pydsa.algorithms.sorting.heap_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Heap-based, sort by building a heap from the array, then repeatedly extracting the maximum element from the heap and inserting it into the sorted array.

pydsa.algorithms.sorting.insertion_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Sort by splitting the array into a sorted and an unsorted part, and values from the unsorted part are picked and placed at the correct position in the sorted part.

pydsa.algorithms.sorting.is_sorted(arr, /, *, key=<function <lambda>>, reverse=False)¶

Check whether the list is sorted.

pydsa.algorithms.sorting.merge_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Sort by dividing the array into two halves (divide-and-conquer), calling itself for the two halves, and then merging the two sorted halves.

pydsa.algorithms.sorting.odd_even_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

A variation of Bubble Sort. The algorithm runs until the array elements are sorted and in each iteration two phases occurs – odd and even phases.

pydsa.algorithms.sorting.pigeonhole_sort(arr, /, *, key=<function <lambda>>, reverse=False, sorting_algorithm=<function bubble_sort>)¶

Modified counting sort that sorts the range using an underlying algorithm, e.g., quicksort. This modified algorithm works for any type of elements.

pydsa.algorithms.sorting.proxmap_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Sort by partitioning an array of data items, or keys, into several buckets. The name is short for computing a ‘proximity map’ which indicates for each key K the beginning of a subarray where K will reside in the final sorted order. Keys are placed into each subarray using Insertion sort.

pydsa.algorithms.sorting.quicksort(arr, /, *, key=<function <lambda>>, reverse=False, pivot_choosing_algorithm=None)¶

Sort by picking an element as pivot and partitions the given array around the picked pivot (divide-and-conquer). Pivot choosing algorithm is a function that takes an array and returns a pivot. If it is not provided, median-of-three algorithm is used.

pydsa.algorithms.sorting.radix_sort(arr, /, *, key=<function <lambda>>, reverse=False, order='MSD')¶

Sort by grouping the individual digits of the same place value. Then, sort the elements according to their increasing/decreasing order.

pydsa.algorithms.sorting.selection_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Sort by repeatedly finding the minimum/maximum element from unsorted part and move it to the front.

pydsa.algorithms.sorting.sleep_sort(arr, /, *, key=<function <lambda>>, reverse=False, amplify=1.0)¶

Sort by starting a separate task for each item to be sorted, where each task sleeps for an interval corresponding to the item’s sort key, then emits the item.

pydsa.algorithms.sorting.slowsort(arr, /, *, key=<function <lambda>>, reverse=False)¶

SlowSort is an example of MultiplyAndSurrender – the worst possible sort algorithm. The algorithm decompose the problem of sorting n numbers in ascending order into: 1. Finding the maximum of those numbers, and 2. Sorting the remaining ones.

Sub-problem 1 can be further decomposed into: 1.1 find the maximum of the first n/2 elements, and 1.2 find the maximum of the remaining n/2 elements, and 1.3 find the largest of those two maxima.

pydsa.algorithms.sorting.stooge_sort(arr, /, *, key=<function <lambda>>, reverse=False)¶

Sort by dividing the array into two overlapping parts (2/3 each). Then it performs sorting in the first 2/3 parts, and then it performs sorting in the last 2/3 part. After that, sorting is done again on the first 2/3 part to ensure the array is sorted.

pydsa.algorithms.sorting.worstsort(arr, /, *, key=<function <lambda>>, reverse=False, recursion_depth=1, sorting_algorithm=<function bubble_sort>)¶

For k equals to 0, Worstsort use Bubble sort to sort the array. Otherwise, the algorithm first generates a list of all permutations of the array. Then, it uses Bubble sort to sort the list of permutations and returns the first element to worstsort(arr, k-1).
